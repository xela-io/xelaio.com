Die erste Tasse Kaffee ist kein Getränk. Sie ist ein Ritual. Wasser kochen, Mühle einstellen, aufgießen, warten. Vier Minuten, in denen der Kopf langsam hochfährt.

## Der Mythos vom 10x-Developer

Es gibt keine 10x-Developer. Es gibt Leute, die gelernt haben, Ablenkungen zu eliminieren. Die erste Stunde am Morgen — Kaffee, Stille, ein offenes Problem — ist produktiver als der gesamte Nachmittag mit Meetings.

## Flow braucht Anlauf

Das Schwierige am Deep Work ist nicht die Arbeit selbst. Es ist der Übergang. Slack schließen, Notifications aus, den Kontext laden. Das dauert 15–20 Minuten. Jede Unterbrechung setzt diesen Timer zurück.

> Der beste Code entsteht nicht, wenn man schnell tippt. Er entsteht, wenn man lange genug nachgedacht hat, um wenig tippen zu müssen.

## Nachmittagskaffee

Die zweite Tasse ist anders. Keine Ruhe, kein Ritual. Eher ein Rettungsanker gegen das Post-Lunch-Tief. Funktioniert trotzdem.

Manchmal frage ich mich, ob ich Code schreibe, weil ich Kaffee trinke — oder Kaffee trinke, weil ich Code schreibe.
